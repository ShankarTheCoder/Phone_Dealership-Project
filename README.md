# Phone_Dealership-Project
# 📱 Smartphone Purchase System (Java OOP Project)

## 📖 Project Overview

The **Smartphone Purchase System** is a simple Java-based project developed using **Object-Oriented Programming (OOP)** concepts. This project simulates a real-world mobile shop scenario where customers attempt to purchase smartphones, and employees manage the selling process.

If a customer does not have enough money to buy a smartphone, the system provides an **EMI (Equated Monthly Installment)** option.

---

## 🚀 Features

* 📦 Create and manage smartphone objects
* 👤 Customer interaction with the system
* 🧑‍💼 Employee handles phone selling process
* 💰 Purchase decision based on available cash
* 📊 EMI calculation for insufficient funds
* 🖨️ Clean output using `toString()` method

---

## 🧩 Technologies Used

* Java
* Object-Oriented Programming (OOP)

---

## 🏗️ Project Structure

The project consists of the following classes:

### 1. Smartphone

* Stores smartphone details such as model name, price, and color

### 2. Customer

* Represents a customer with name and available cash
* Can attempt to buy a smartphone

### 3. Employee

* Handles the selling process
* Checks if the customer can afford the phone
* Provides EMI option if needed

### 4. Practice (Main Class)

* Entry point of the program
* Creates objects and simulates real scenarios

---

## ⚙️ How It Works

1. A customer tries to buy a smartphone
2. The employee checks the customer's available cash
3. If the customer has enough money:

   * The smartphone is sold
4. If not:

   * EMI option is calculated and displayed
